# 第2章 创业的记分牌
* 数据分析的意义：在资金耗尽以前，找到正确的产品和市场。
## 数据指标设定
* 评价标准
  - 比较性强的。比如“本周用户转化率比上周高”
  - 通俗易懂。类比为“奥卡姆剃刀”
  - 是一个比率。其有点在于
    - 可操作性强，是行动的向导
    - 天生的比较性指标
    - 适用于比较各种因素间的相生和相克（正相关和负相关）
  - 会改变行为。比如指导后续的营销动作。
* 服务于同一目标，即留住用户，促进口碑传播，有效获取新用户，或者创造营收。反例就是如果指标与目标不想关，员工的商业行为就不会随之发生改变，纯纯浪费时间。
* 另一个关注点在于指标的耦合性，比如转化率与购买所需时间等

## 数据指标的不同划分方式
* 首先，分析师应该关心那些推动公司业绩增长的指标，即关键绩效指标（KPI）
* 划分分类
  - 定性指标与定量指标：定性指标带有主观因素，而定量指标排斥主观因素；定性指标用于回答“为什么”，定量指标回答“多少”。
  - 虚荣指标和可付诸行动指标
    - 简易评判标准：面对“依据这个指标，我将如何改变当前的商业行为”这一问题，如果回答不上来，则不必纠结这个指标。
    - 常见虚荣指标（不一定完全正确）：
      - 点击量
      - 页面浏览量
      - 访问量
      - 独立访客数
      - 粉丝/好友/点赞数
      - 网站停留时间/浏览页数
      - 收集到的用户邮箱地址数量
      - 下载量
  - 探索性指标和报告性指标
    |  | 我们知道 | 我们不知道 |
    |---|---|---|
    | 我们知道的 | 是可能并不为真的事实，需经由数据的检验（检验手头上的事实和假设，以确保不是自欺欺人） | 是直觉，需要我们评估并训练以提高其效能和效率（验证直觉，把假设变成证据） |
    | 我们不知道的 | 可通过调研解答的问题，可使之成为我们的行为基准并流程化（为业务预测、瀑布式开发流程图和董事会议提供数据） | 是探索，蕴含着我们自身独特的优势，能带来有趣的顿悟（帮助我们发现黄金机遇） |
    - 案例：“妈妈圈”成功之路 -- 发掘高活跃用户的共同特征 -> 在一个更小、更容易的目标市场中培养更多具有黏性的高活跃度用户
  - 先见性数据指标和后见性数据指标：预测未来与解释过去
  - 相关性指标和因果性指标
    - 误区：错把相关性当成因果性
    - 相关性帮助预测未来，因果性用来改变未来
* 最后，数据指标并非一成不变的，要及时调整

## 测试是精益数据分析的灵魂
* 方法论
  - 市场细分：分析一群拥有某种共同特征的人
  - 同期群分析：比较相似群体随时间的变化
    - 适用场景：营收、客户流失率、口碑的病毒式传播、客户支持成本等
  - A/B测试
  - 多变量分析：改动产品多方，看哪个与结果相关性最大

## 精益数据分析周期
* 核心点：如何找到一个有意义的指标，然后通过试验验收改善它，直到令你满意。

# 第4章 以数据为导向与通过数据获取信息
* 数据很强大，但是不要陷入过度分析误区。
  - 由于AI更多是找到局部最优点，坚持人类提供灵感，机器负责实现/验证
  - 数据是验证假设的极佳工具。
* 创业者需要避免的数据圈套：
  - 假设数据没有噪声
  - 忘记归一化
  - 排除异常点
  - 包括异常点
  - 忽视季节性
  - 抛开基数谈增长
  - 数据呕吐：不知道什么数据对你更重要
  - 谎报军情的指标
  - “不是在这儿收集的”综合征
  - 关注噪声
* 总结来说，精益创业不等于不假思索地重复“开发->测试->认知”循环，而在于真正理解发生了什么并接受新的可能性
